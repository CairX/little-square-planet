Prototype: Unity

2017-03-26

Iteration 1

Unity Executable

What Works

  • Watching the seeds grow, because it is satisfying visual feedback of progress
  • Different blooming patterns, because you can express yourself
  • Competitive clicking, because self-imposed time-pressure makes it a game (not our aim)

What's Lacking

  • No reward, because it feels pointless
  • No sense of persistence, because progress is lost on exit
  • Avatar, because you feel like a world-overlord rather than a world-participant (immersion)

Proposed Changes

  • Seamless tiles, because visible grid breaks the illusion of choice
  • Encapsulated island, because the current version feels endless and thus constrained
  • Increased growth time, because a short growth time turns the game into a race
  • Keyboard instead of mouse, because it feels more immersive with an avatar
  • Counter, because no feedback makes everything feel pointless
  • Inventory/economy, to create a purpose and show progress

Iteration 2

Unity Executable

Result of Changes

Seamless Tiles

The change made the borders invisible, which made placement of seeds difficult. Thus, we want to be able to see the borders, but have them be discrete.

Encapsulated Island

It works because it distinguishes the planting area.

Increased Growth Time

It is still too short for the intended game. For testing purposes, it is adequate, but requires more testing with an exposed variable.

Counter/Economy

Creates a sense of progression.

Keyboard Instead of Mouse

Was never implemented.
